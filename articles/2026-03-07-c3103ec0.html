Chardet dispute shows how AI will kill software licensing • The Register

Chardet の紛争は、AI がソフトウェアライセンスを破壊する方法を示す

Earlier this week, Dan Blanchard, maintainer of a Python character encoding detection library called chardet, released a new version of the library under a new software license. Version 7.0 employs an MIT license in place of the previous GNU Lesser General Public License (LGPL). Developers who have an eye on commercial use of their open source works tend to prefer permissive licenses like MIT’s because they impose fewer obligations than copyleft licenses like GPL/LGPL, which require derivative works to be distributed under the same terms. 「ソフトウェア開発の経済は終わり、消え、終焉、終わりだ！」 Blanchardは、彼がAI（AnthropicのClaude）を使用して実現したclean room実装であると主張したため、彼がライセンスを変更する権利があると言明しました。Claudeはchardetのコードを学習したか、もしそうであれば、Claudeの出力が学習データを複製したかどうかは不明です。 Mark Pilgrim氏と名乗る個人は、ライブラリの元の作成者であると主張し、プロジェクトのGitHubリポジトリで問題を開きました。彼はBlanchardがソフトウェアのライセンスを変更する権利がないと主張し、LGPLの要件を引用しました。LGPLの要件によれば、修正されたコードは同じLGPLライセンスのもとでリリースする必要があります。 "修正されたライセンスコードは同じLGPLライセンスのもとでリリースする必要があります。" と名乗る人物は言いました。「メンテナの主張である「完全な再実装」は無関係です、なぜなら、コードが完全に再実装されたとしても、LGPLライセンスの要件は変更されないからです。」
